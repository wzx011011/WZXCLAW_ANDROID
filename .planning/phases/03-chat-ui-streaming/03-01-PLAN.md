```dart
enum MessageRole { user, assistant, tool }

enum ToolCallStatus { running, done }

class ChatMessage {
  final int? id;           // SQLite row id, null before first insert
  final MessageRole role;
  final String content;    // text content for user/assistant, tool name for tool
  final String? toolName;  // only for role==tool
  final ToolCallStatus? toolStatus; // only for role==tool
  final DateTime createdAt;

  // For the streaming assistant message that is still being built:
  final bool isStreaming;

  ChatMessage({
    this.id,
    required this.role,
    required this.content,
    this.toolName,
    this.toolStatus,
    required this.createdAt,
    this.isStreaming = false,
  });

  ChatMessage copyWith({
    int? id,
    String? content,
    ToolCallStatus? toolStatus,
    bool? isStreaming,
  }) => ChatMessage(
    id: id ?? this.id,
    role: role,
    content: content ?? this.content,
    toolName: toolName,
    toolStatus: toolStatus ?? this.toolStatus,
    createdAt: createdAt,
    isStreaming: isStreaming ?? this.isStreaming,
  );

  Map<String, dynamic> toDbMap() => {
    'role': role.index,
    'content': content,
    'tool_name': toolName,
    'tool_status': toolStatus?.index,
    'created_at': createdAt.millisecondsSinceEpoch,
  };

  factory ChatMessage.fromDbMap(Map<String, dynamic> map) => ChatMessage(
    id: map['id'] as int?,
    role: MessageRole.values[map['role'] as int],
    content: map['content'] as String,
    toolName: map['tool_name'] as String?,
    toolStatus: map['tool_status'] != null
        ? ToolCallStatus.values[map['tool_status'] as int]
        : null,
    createdAt: DateTime.fromMillisecondsSinceEpoch(map['created_at'] as int),
  );
}
```

Create `lib/services/chat_store.dart` -- the central state manager that the UI subscribes to.

ChatStore is a singleton (following ConnectionManager pattern) that:
1. Subscribes to `ConnectionManager.instance.messageStream`
2. Accumulates streaming events into a live ChatMessage
3. Persists completed messages to ChatDatabase
4. Exposes reactive streams for the UI

Implementation details:

```dart
class ChatStore {
  static final ChatStore _instance = ChatStore._();
  static ChatStore get instance => _instance;
  ChatStore._() { _init(); }

  // -- Reactive state --
  final _messagesController = StreamController<List<ChatMessage>>.broadcast();
  Stream<List<ChatMessage>> get messagesStream => _messagesController.stream;

  final _streamingController = StreamController<bool>.broadcast();
  Stream<bool> get streamingStream => _streamingController.stream;

  // -- Internal state --
  final List<ChatMessage> _messages = [];
  ChatMessage? _streamingMessage;  // the in-progress AI message
  bool _isStreaming = false;
  StreamSubscription? _wsSubscription;

  bool get isStreaming => _isStreaming;
  List<ChatMessage> get messages => List.unmodifiable(_messages);
  // Full display list: messages + streaming message appended if active
  List<ChatMessage> get displayMessages {
    if (_streamingMessage != null) {
      return [..._messages, _streamingMessage!];
    }
    return List.unmodifiable(_messages);
  }
}
```

Key methods:
- `_init()`: Subscribe to `ConnectionManager.instance.messageStream`. On each WsMessage, call `_handleWsMessage(event, data)`.
- `_handleWsMessage(event, data)`:
  - `stream:text_delta`: If no _streamingMessage, create one with empty content, isStreaming=true. Append data (the text delta) to _streamingMessage.content. Set _isStreaming=true.
  - `stream:tool_use_start`: Create a new ChatMessage with role=tool, toolName=data['name'] or data.toString(), toolStatus=running. Insert into _messages list. Persist to DB. If there's a _streamingMessage, also insert it into _messages and persist before the tool message, then clear _streamingMessage (tool calls can interrupt text streaming).
  - `stream:done`: If _streamingMessage exists, mark isStreaming=false, add to _messages, persist to DB, clear _streamingMessage. Set _isStreaming=false.
  - `stream:error`: Same as stream:done but append error text.
  - `message:assistant`: If not currently streaming, create a complete assistant message from data['content'] and add to _messages.
  - `message:user`: Echo of user message -- ignore (we already added it locally on send).
  - `session:messages`: Bulk history sync. Clear _messages, parse data as list of {role, content} maps, create ChatMessage for each, replace _messages, persist all. Notify listeners.
- `sendMessage(String text)`: Create a ChatMessage(role=user, content=text). Add to _messages. Persist to DB. Send via ConnectionManager as `WsMessage(event: WsEvents.commandSend, data: {'content': text})`. Notify listeners.
- `stopGeneration()`: Send `WsMessage(event: WsEvents.commandStop)`. If _streamingMessage, finalize it (isStreaming=false), add to _messages, persist, clear.
- `clearSession()`: Call ChatDatabase.instance.clearAll(). Clear _messages. Clear _streamingMessage. Set _isStreaming=false. Notify listeners. Per D-10.
- `loadHistory()`: Query ChatDatabase.instance.getMessages(limit: 100). Set _messages. Notify listeners. Per D-11.
- `loadMoreMessages()`: Query ChatDatabase with offset=_messages.length, limit=100. Prepend to _messages. Notify listeners. Per D-11.
- `_notifyListeners()`: Add current displayMessages to _messagesController, add _isStreaming to _streamingController.
- `dispose()`: Cancel subscription, close controllers.

Import: dart:async, chat_message.dart, chat_database.dart, connection_manager.dart, ws_message.dart.

IMPORTANT: For streaming text accumulation, concatenate deltas as plain strings. The streaming message's content grows incrementally: `_streamingMessage = _streamingMessage!.copyWith(content: _streamingMessage!.content + delta)`.
