Semantic status colors (inherited from existing patterns):

| Status | Color | Usage |
|--------|-------|-------|
| Idle (not recording) | `Colors.white38` | Mic button default state -- matches send button disabled color |
| Recording (active) | `Colors.redAccent` | Mic button while user is pressing -- visual indicator that recording is active |
| Disabled (disconnected) | `Colors.white24` | Mic button when WebSocket is disconnected -- matches send button disabled state |

Mic button color logic mirrors the send button pattern established in Phase 3:
- Connected + idle: `Colors.white38` (same as send button's `Colors.white24` but slightly brighter so the mic icon remains discoverable)
- Connected + recording: `Colors.redAccent` (clear visual feedback that audio is being captured)
- Disconnected: `Colors.white24` (matches send button disabled color)

Accent reserved for: send button enabled state only. Mic button does NOT use accent color.

## Interaction Contracts

### Mic Button Placement
- Position: right side of the TextField, between the text field and the send/stop button
- Preceded by `SizedBox(width: 8)` (same spacing as the existing gap before send button)
- Standard Material `IconButton` with no background, consistent with existing send button style

### Idle State (Default)
- Icon: `Icons.mic` (Material Icons microphone outline)
- Color: `Colors.white38`
- Tooltip: "语音输入"
- Button is enabled when connected, disabled when disconnected
- Size: default `IconButton` size (48px touch target, 24px icon)

### Recording State (While User is Pressing)
- Triggered on: `onLongPressStart` gesture event
- Icon: `Icons.mic` with a pulsing opacity animation (0.5 to 1.0, 800ms cycle) to indicate active recording
- Color: `Colors.redAccent`
- Haptic feedback: `HapticFeedback.mediumImpact()` on press start (one-time, not repeated)
- Button remains visually pressed while finger is down

### Releasing (End of Recording)
- Triggered on: `onLongPressEnd` gesture event
- Animation stops immediately, button returns to idle state
- If recognition succeeds: recognized text is set as `_inputController.text` (replaces any existing input)
- If recognition fails: SnackBar shows error message for 2 seconds, input field is unchanged
- User can then edit the text and send via the existing send button

### Connection Dependency
- Mic button follows the same connection guard as the send button:
  - `onLongPressStart` is only active when `isConnected == true`
  - When disconnected: button color is `Colors.white24`, no gesture response
- This is consistent with how the send button is disabled when disconnected

### Streaming Guard
- When `_isStreaming == true`, the stop button is shown and the send button is hidden
- The mic button remains visible during streaming but is visually de-emphasized (disabled state appearance)
- User should stop generation before using voice input (no explicit enforcement -- the mic button simply does not respond during streaming)
- Rationale: do not add voice input while streaming is active; the UI naturally guides the user to stop first

### TextField Behavior After Voice Recognition
- Recognized text replaces any existing text in `_inputController` (not appended)
- Cursor moves to end of recognized text
- TextField remains focused so user can immediately type corrections or additions
- The existing `onSubmitted` callback works normally -- user can press the keyboard send action

## Layout Specification

### Input Bar Layout (Updated)

The existing input bar `_buildInputBar()` layout changes from:

```
[TextField (Expanded)] [8px gap] [Send/Stop IconButton]
```

To:

```
[TextField (Expanded)] [8px gap] [Mic IconButton] [8px gap] [Send/Stop IconButton]
```

Both icon buttons use default `IconButton` sizing (48px touch target, 24px icon canvas). The input bar remains within the same container with `padding: EdgeInsets.symmetric(horizontal: 8, vertical: 6)` and `background: #16213E`.

### Mic Button Dimensions
- Touch target: 48x48px (Material default for `IconButton`)
- Icon size: 24px (Material default)
- No background shape, no border -- matches existing send button style

### Recording Visual Feedback
- The mic icon itself pulses (opacity animation), NOT a separate overlay or background
- No ripple effect on long-press (to avoid visual confusion with tap gestures)
- No waveform or amplitude visualization (per CONTEXT.md: "minimal audio feedback")

## Animation Specification

### Mic Button Recording Pulse
- Property: `opacity`
- Range: 0.5 to 1.0
- Duration: 800ms
- Curve: `Curves.easeInOut`
- Repeat: infinite (reversing, so smooth fade in/out)
- Trigger: `onLongPressStart` begins the animation
- End: `onLongPressEnd` stops the animation, opacity snaps back to 1.0

Implementation approach: Use `AnimationController` with `repeat(reverse: true)` inside a `StatefulWidget` for the mic button, with `SingleTickerProviderStateMixin`. This follows the same pattern as `_StreamingCursor` in `home_page.dart`.
