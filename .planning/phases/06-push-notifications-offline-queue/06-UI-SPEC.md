## Phase 6 Specific UI Elements

This phase is primarily service-layer work. The Flutter UI changes are minimal:

### 1. Push Notification Toggle (Settings Page)

Location: `lib/pages/settings_page.dart` -- add below the connection state display.

Component: `SwitchListTile` widget.

| Property | Value |
|----------|-------|
| Title text | "推送通知" |
| Subtitle text | "AI 任务完成时发送通知" |
| Active track color | `#6366F1` (accentColor) |
| Inactive track color | `Colors.white24` |
| Active thumb color | `Colors.white` |
| Default state | `true` (push enabled by default) |
| Persistence | `SharedPreferences` key `push_notifications_enabled` |

Visual spec: Match the existing card style -- wrap in a `Container` with `color: surfaceColor (0xFF16213E)`, `borderRadius: 8`, `padding: EdgeInsets.all(12)`. Or use a standalone `SwitchListTile` placed below the connection state card with the same 24px vertical spacing.

Spacing: 24px gap below the connection state card (matches existing `SizedBox(height: 24)` pattern).

### 2. Offline Sync Banner (Home Page)

Shown briefly when the app reconnects and queued messages are being synced from the relay.

| Property | Value |
|----------|-------|
| Widget | `SnackBar` (matches existing voice error pattern) |
| Background color | `Color(0xFF16213E)` (surfaceColor) |
| Text color | `Colors.white70` |
| Content text | "正在同步离线消息..." |
| Duration | 3 seconds |
| Behavior | `SnackBarBehavior.floating` |

No persistent indicator needed. The user sees messages appear in the chat list as they sync -- that is the implicit feedback.

### 3. Android System Notification (Non-Flutter UI)

The notification itself is rendered by the Android system, not Flutter. Contract for the notification payload:

| Property | Value |
|----------|-------|
| Channel: task_complete | Importance: default, sound: default |
| Channel: error | Importance: high, sound: default |
| Small icon | `@mipmap/ic_launcher` (app icon) |
| Title format | `wzxClaw` |
| Body format | `{agent_name} completed: {first 50 chars of last message}` |
| Grouping | By session (same session notifications collapse into summary) |
| Tap action | Navigate to `/` (HomePage) and scroll to latest message |
| Suppression | Do NOT show if app is in foreground (check via `WidgetsBindingObserver`) |

### 4. Empty State -- No Additional Empty States

Phase 6 does not introduce new empty states. The existing "暂无消息" empty state in the chat list remains unchanged.

## Interaction Contracts

### Push Toggle Interaction
- Toggle ON: Immediately subscribe to FCM topic / register token. Save `true` to SharedPreferences. No visual feedback beyond switch animation.
- Toggle OFF: Immediately unsubscribe / unregister token. Save `false` to SharedPreferences. No visual feedback beyond switch animation.
- No confirmation dialog needed (low-risk reversible action).

### Notification Tap Interaction
- Tap notification while app is in background: App resumes, navigates to `/` (HomePage), auto-scrolls to bottom to show latest message.
- Tap notification while app is killed: Cold start, navigate to `/`, display synced messages including the one that triggered the notification.

### Offline Sync Interaction
- On reconnect (WebSocket state transitions to `connected`): Check relay for queued messages. If messages exist, show sync SnackBar. Messages appear in chat list incrementally as they arrive. No loading spinner needed -- messages stream in naturally.
- If no queued messages: No SnackBar, no UI feedback. Silent reconnect (existing behavior).
