    **relay/Dockerfile:**
    Use multi-stage build for minimal image:
    - Stage 1 (builder): `FROM node:20-alpine AS builder`, WORKDIR /app, COPY package*.json, RUN npm ci --production
    - Stage 2 (runtime): `FROM node:20-alpine`, WORKDIR /app, COPY --from=builder /app/node_modules ./node_modules, COPY . .
    - Set NODE_ENV=production
    - EXPOSE 8080 (internal port)
    - USER node (non-root for security)
    - CMD ["node", "server.js"]

    **relay/docker-compose.yml:**
    ```yaml
    version: "3.8"

    services:
      relay:
        build:
          context: .
          dockerfile: Dockerfile
        container_name: wzxclaw-relay
        restart: unless-stopped
        environment:
          - AUTH_TOKEN=${AUTH_TOKEN}
          - PORT=8080
        networks:
          - wzxclaw-net
        # No ports exposed to host -- nginx connects via Docker network

    networks:
      wzxclaw-net:
        external: true
        # If the network doesn't exist on NAS, user creates it:
        # docker network create wzxclaw-net
        # Then add the nginx container to this network too
    ```

    Note: If wzxclaw-net does not exist as external, document the alternative: use a named network without `external: true` and ensure nginx is on the same compose file or joined to the same network.

    Actually, since this is a personal NAS with existing nginx, make the network simpler:
    - Use a bridge network named `wzxclaw-net`
    - Document that the nginx container needs to be on this network (or use `network_mode: host` for simplicity on NAS)

    Revised docker-compose.yml: Use `network_mode: bridge` with a named network, or simply expose port 8080 to localhost only (`127.0.0.1:8080:8080`) so nginx on the same host can proxy to it. This is simpler for a NAS setup.

    Final docker-compose.yml:
    ```yaml
    version: "3.8"

    services:
      relay:
        build:
          context: .
          dockerfile: Dockerfile
        container_name: wzxclaw-relay
        restart: unless-stopped
        ports:
          - "127.0.0.1:8081:8080"
        environment:
          - AUTH_TOKEN=${AUTH_TOKEN}
          - PORT=8080
    ```

    Port 8081 on localhost to avoid conflict with any existing service on 8080. The relay is only accessible to nginx on localhost, not exposed to the internet directly.

    **relay/.dockerignore:**
    ```
    node_modules
    test
    .git
    .gitignore
    *.md
    nginx/
    ```

    **relay/nginx/relay.conf:**
    Nginx location block snippet to be included in the existing nginx config for 5945.top:
    ```nginx
    # wzxClaw WebSocket Relay
    location /relay/ {
        proxy_pass http://127.0.0.1:8081/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 86400s;
        proxy_send_timeout 86400s;
    }
    ```

    Key details:
    - `proxy_pass http://127.0.0.1:8081/` -- trailing slash strips /relay/ prefix so relay server sees `/`
    - `proxy_http_version 1.1` + Upgrade headers -- required for WebSocket
    - `proxy_read_timeout 86400s` -- 24 hour timeout to prevent nginx from closing long-lived WebSocket connections
    - This snippet should be placed inside the existing `server { ... }` block for 5945.top
