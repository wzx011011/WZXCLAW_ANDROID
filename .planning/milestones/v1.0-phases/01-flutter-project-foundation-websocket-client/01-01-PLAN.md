## Desktop wzxClaw WebSocket Protocol (from src/mobile-client/app.js)

**Connect:** `ws(s)://{host}:{port}/?token={token}`

**Send messages (client -> server):**
```json
{ "event": "command:send", "data": { "content": "user text here" } }
{ "event": "command:stop" }
```

**Receive messages (server -> client), event field:**
- `connected` -- connection confirmed
- `message:user` -- data.content has user message
- `message:assistant` -- data.content has full assistant message
- `stream:text_delta` -- data.content has incremental text chunk
- `stream:tool_use_start` -- data.name has tool name
- `stream:done` -- generation complete
- `stream:error` -- data.error has error message
- `session:messages` -- data is array of {role, content} for full history sync

**Auth:** Token passed as URL query param `?token=`. The desktop server validates this token.

Create `lib/services/connection_manager.dart` implementing a singleton `ConnectionManager` class. This is the most critical file in Phase 1 -- it must handle all connection lifecycle concerns correctly. Reference PITFALLS.md Pitfalls 1, 4, 5, and 6.

**Class structure:**
```dart
class ConnectionManager with WidgetsBindingObserver {
  // Singleton
  static final ConnectionManager _instance = ConnectionManager._();
  static ConnectionManager get instance => _instance;
  ConnectionManager._();

  // Public state
  final _stateController = StreamController<WsConnectionState>.broadcast();
  Stream<WsConnectionState> get stateStream => _stateController.stream;
  WsConnectionState _state = WsConnectionState.disconnected;

  final _messageController = StreamController<WsMessage>.broadcast();
  Stream<WsMessage> get messageStream => _messageController.stream;

  // Internal
  WebSocketChannel? _channel;
  int _reconnectAttempt = 0;
  Timer? _heartbeatTimer;
  Timer? _heartbeatTimeoutTimer;
  Timer? _idleTimer;
  Timer? _reconnectTimer;
  DateTime? _lastMessageTime;
  int _connSeq = 0;  // Connection sequence number to ignore stale callbacks
  final List<String> _sendQueue = [];  // Queued messages as JSON strings
}
```

**Required methods:**

1. `void connect(String url)` -- Connect to the given WebSocket URL (e.g., `ws://192.168.1.100:3000/?token=xxx`). Sets state to `connecting`, creates `WebSocketChannel.connect(Uri.parse(url))`, listens to the channel stream. Increments `_connSeq` to invalidate any stale timers.

2. `void disconnect()` -- Clean disconnect: cancel all timers, close channel with normal close code, set state to `disconnected`, reset `_reconnectAttempt` to 0, clear `_sendQueue`.

3. `void send(WsMessage message)` -- If state is `connected` AND heartbeat is healthy, send immediately via channel. Otherwise, add JSON string to `_sendQueue` (if queue size < maxQueueSize, discard oldest if over limit). When connection restores, flush queue.

4. `void _onMessage(dynamic data)` -- Parse incoming data as JSON. If `event == "pong"`, record pong received, cancel timeout timer, update `_lastMessageTime`. If any other event, update `_lastMessageTime` and add to `_messageController`.

5. `void _startHeartbeat()` -- Start a periodic Timer at `heartbeatInterval` (15s). On each tick: send `{ "event": "ping" }`, start a one-shot Timer at `heartbeatTimeout` (8s). If the timeout fires before pong arrives, call `_forceReconnect("heartbeat timeout")`.

6. `void _stopHeartbeat()` -- Cancel both heartbeat and timeout timers.

7. `void _startIdleMonitor()` -- Start a periodic check (every 10s) that checks if `_lastMessageTime` is more than `maxIdleTime` (60s) ago. If so, force reconnect. This is a secondary guard against silent stalls.

8. `void _forceReconnect(String reason)` -- Log the reason, cancel all timers, close channel with abnormal close code, transition to `reconnecting`, schedule reconnect with exponential backoff.

9. `void _scheduleReconnect()` -- Calculate delay: `min(reconnectMaxDelay, reconnectBaseDelay * 2^_reconnectAttempt) + random(0, jitterMaxMs)`. Schedule `_reconnectTimer`. Increment `_reconnectAttempt`. Set state to `reconnecting`.

10. `void _flushQueue()` -- On successful connect (after receiving `connected` event or after state transitions to `connected`): send all queued messages in order, clear the queue.

11. `void _onChannelDone()` -- If not intentionally disconnected, schedule reconnect.

12. `void _onChannelError(Object error)` -- Log error, schedule reconnect.

**Lifecycle handling (WidgetsBindingObserver):**
- In constructor, add `WidgetsBinding.instance.addObserver(this)`.
- Override `didChangeAppLifecycleState(AppLifecycleState state)`:
  - On `paused`/`inactive`: stop heartbeat timers, keep channel alive (Android will kill it anyway).
  - On `resumed`: force-close existing channel (do NOT trust it), reconnect with `_connSeq++` to invalidate stale callbacks, start fresh heartbeat after connect.

**Connection sequence pattern (prevents stale callbacks):**
```dart
void connect(String url) {
  final seq = ++_connSeq;
  _url = url;
  _setState(WsConnectionState.connecting);
  // ... create channel ...
  channel.stream.listen(
    (data) {
      if (seq != _connSeq) return; // stale connection, ignore
      _onMessage(data);
    },
    onDone: () {
      if (seq != _connSeq) return;
      _onChannelDone();
    },
    onError: (error) {
      if (seq != _connSeq) return;
      _onChannelError(error);
    },
  );
}
```

**State transition rules:**
- `disconnected` -> `connecting`: on connect()
- `connecting` -> `connected`: on first message received (or `connected` event from server)
- `connecting` -> `reconnecting`: on error/done before receiving any message
- `connected` -> `reconnecting`: on heartbeat timeout, idle timeout, channel done/error
- `reconnecting` -> `connecting`: on scheduled reconnect attempt
- `reconnecting` -> `disconnected`: on disconnect()
- Any state -> `disconnected`: on disconnect()

**Important implementation notes:**
- Use `_stateController.add(_state)` to broadcast state changes.
- The `_messageController` broadcasts ALL non-pong messages to subscribers.
- On successful connection (state -> connected), start heartbeat AND idle monitor.
- On any disconnect/reconnect, stop heartbeat AND idle monitor.
- Store the URL so that reconnect can use the same URL without requiring it again.
- Use `WebSocketChannel.connect(Uri.parse(url))` from `web_socket_channel` package.
