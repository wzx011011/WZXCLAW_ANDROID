    1. Change the server URL hint text from `'ws://192.168.1.100:3000'` to `'wss://5945.top/relay/'` to guide users toward the relay URL format.

    2. Update the `_connect()` method to construct URLs with role=mobile. The new logic:
       ```dart
       void _connect() {
         final serverUrl = _serverUrlController.text.trim();
         final token = _tokenController.text.trim();
         if (serverUrl.isEmpty) return;

         _saveValues();

         // Build URL with token and role parameters.
         // Supports both relay URL (wss://host/relay/) and direct URL (ws://host:port).
         final uri = Uri.parse(serverUrl);
         final params = Map<String, String>.from(uri.queryParameters);
         params['role'] = 'mobile';
         if (token.isNotEmpty) {
           params['token'] = token;
         }
         final fullUrl = uri.replace(queryParameters: params).toString();
         ConnectionManager.instance.connect(fullUrl);
       }
       ```

       This approach:
       - Works with relay URL: `wss://5945.top/relay/` becomes `wss://5945.top/relay/?role=mobile&token=XXX`
       - Works with direct URL: `ws://192.168.1.100:3000` becomes `ws://192.168.1.100:3000?role=mobile&token=XXX`
       - Backward compatible: if user already has a URL with query params, they are preserved
