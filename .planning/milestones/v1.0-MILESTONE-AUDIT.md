## 4. API/Message Coverage

All 15 WebSocket event types have proper producers and consumers:

- **Outgoing:** command:send (ChatStore, ProjectStore), command:stop (ChatStore), fcm:register (NotificationService), ping (ConnectionManager)
- **Incoming consumed:** connected (ConnectionManager state), message:assistant (ChatStore + ProjectStore), stream:text_delta (ChatStore streaming), stream:tool_use_start (ChatStore tool badge), stream:done (ChatStore finalize + Relay queue+push), stream:error (ChatStore error + Relay queue+push), session:messages (ChatStore bulk sync)
- **System events:** system:desktop_disconnected (ConnectionManager triggers reconnect), system:mobile_disconnected (desktop side)
- **Consumed/filtered:** message:user (ChatStore ignores echo), ping/pong (room.js consumed without forwarding), fcm:register (room.js consumed without forwarding)

## 6. E2E Flow Traces

### Flow 1: Connect and Chat -- COMPLETE (13 steps)
HomePage.initState -> autoConnect (SharedPreferences) -> ConnectionManager.connect -> Relay auth+join -> connected state -> StreamBuilder green dot -> user sends message -> ChatStore.sendMessage -> ConnectionManager.send -> Relay forwards -> desktop responds stream:text_delta -> ChatStore accumulates -> HomePage renders streaming text -> desktop sends stream:done -> ChatStore persists to SQLite

### Flow 2: Voice Input to Chat -- COMPLETE (8 steps)
User long-presses MicButton -> VoiceInputService.startListening -> SpeechToText.listen -> user releases -> stopListening -> onResult -> _inputController.text = text -> user edits -> ChatStore.sendMessage

### Flow 3: Project Switch -- COMPLETE (9 steps)
User taps hamburger -> ProjectDrawer opens -> auto fetchProjects -> ConnectionManager sends /projects -> desktop responds -> ProjectStore parses -> StreamBuilder renders list -> user taps project -> ProjectStore.switchProject -> ConnectionManager sends /switch -> Navigator.pop -> desktop responds

### Flow 4: Offline + Push Notification -- COMPLETE (9 steps, 2 require human)
App backgrounded -> heartbeat stops -> connection drops -> Relay queues messages -> fcm.sendPushNotification on task-complete -> [human] FCM delivers to device -> [human] user taps notification -> app opens -> ConnectionManager reconnects -> Relay flushes queue -> HomePage shows sync SnackBar

### Flow 5: Push Toggle -- COMPLETE (5 steps)
User opens Settings -> toggles OFF -> SharedPreferences saves, fcm:register with null sent -> Relay clears fcmToken -> kill+relaunch -> toggle persists -> toggles ON -> real FCM token sent
